# Variant character map

> 权威实现：[`scripts/normalize_variants.py`](../scripts/normalize_variants.py)  
> 流水线位置：`opencc t2s` **之后**、`resegment` **之前**（见 [segment-and-punct.md](segment-and-punct.md)）  
> 统计：单字 **79** 条 + 多字 **1** 条（以脚本为准，本文与脚本同步维护）

## 原则

1. **opencc t2s 不够** — 扩展区（㑹/㸃…）、日新字体（増/従…）、四库旧形（髙/宻…）t2s 常扫不到。  
2. **全活跃库一键洗** — 用户点名时不要只洗当前书（曾漏他书残留）。  
3. **目标字用通行简体** — 洗完后 `repair_false_punct` 词表必须跟简体（`枢密`/`高庆裔`），勿只留 `枢宻`/`髙`。  
4. **用户点名优先入库** — 点一个加一个，并标「用户点名」；跑完 residual=0。  
5. **禁止** 无差别全局 `著→着`（会破坏著作/著名）。  
6. 多字映射 **先于** 单字（`MULTI` 先跑）。

## 用法

```bash
cd /path/to/text-search
python3 scripts/normalize_variants.py                 # 全活跃 data/<时期>/*.txt
python3 scripts/normalize_variants.py data/辽宋夏金/大金国志.txt
python3 scripts/normalize_variants.py --dry-run       # 只统计
```

验收：用户关键键 residual **0**（默认至少 `宻髙畱戸`；`熈` 亦应 0）。

## 与断句/分段的关系

| 步骤 | 说明 |
|------|------|
| 先 variants | 否则 repair 词表简体键对不上 |
| 再 resegment | 卷层/干支/年谱 |
| 再 repair | 专名误断用**简体**词表 |

异体是**字形**问题；`海陵，炀王` / `高。永昌` 是**断句**问题（见 segment-and-punct §3），不要只靠异体表。

---

## 追加流程（再发现异体时）

1. 确认是否应用 opencc 可解 → 不解则入库。  
2. 写入 `VARIANT_MAP` 或 `MULTI`（脚本）。  
3. 用户点名 → 注释 `# 用户点名`，并加入 residual 检查键。  
4. 跑全库 `normalize_variants.py`，residual=0。  
5. 更新**本文** + README 示例表。  
6. 若影响专名（如 髙庆裔），确认 repair 词表已是简体。
