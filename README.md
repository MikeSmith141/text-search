# 史料检索（text-search）

零依赖、可自托管的**中国古典/史料全文检索** Web 应用。  
面向手机与桌面：时期筛选、书级勾选、OR/AND 查询、按史料时代排序、全文阅读器、辽宋金夏等语料入库与清洗流水线。

| 项 | 说明 |
|----|------|
| 运行 | Python 3 标准库 `http.server` + 单页 HTML，**无 pip 依赖** |
| 默认端口 | `8899` |
| 数据布局 | `data/<时期>/<书名>.txt` + `data/books_meta.json` |
| 仓库性质 | **私有**（含整理后的公版/自备文本与元数据；请勿公开分发未经授权的整理本） |

## 功能一览

### 检索与展示

| 功能 | 说明 |
|------|------|
| 全文检索 | 按行扫描 `data/` 下全部活跃 `.txt`；`/` = 任一（OR），`+` = 同时（AND）；空格兼容 AND，`|` 兼容 OR |
| 书名检索 | 只搜书名 / 标签 / 作者 / 时代，一书一卡，可「阅读全文」 |
| 时期筛选 | 固定时期：先秦 → 秦汉 → 魏晋南北朝 → 隋唐五代 → 辽宋金夏 → 元 → 明 → 清 → 民国 |
| 书级筛选 | 「筛选范围」内按书勾选；API：`books=时期/书名,...` |
| 结果排序 | **史料时代**（`books_meta.era`，早→晚）→ 目录时期 → 书名 → 行号（同书目录顺序） |
| 命中预览 | 默认只显示命中行整段；可「展开前后文」 |
| 书组收起 | 单书 **收起本书 / 展开 N 条**；顶栏 **收起全部书 / 展开全部书** |
| 分页加载 | 真实 `total` + `has_more` + **加载更多**（修复 limit 假总数导致后排书「消失」） |
| 标签展示 | `书名·[时代]·作者`（来自 `books_meta.json`） |
| 高亮 | 扁平 `keywords` 高亮；阅读器上一处/下一处用 `terms`（OR 感知） |

### 全文阅读器

- 入口：命中「打开定位」、书组「阅读全书」、筛选列表「阅读」、书名检索「阅读全文」
- 字号 A±、行号跳转、进度条、大书约 400 行窗口加载上文/下文
- **在本书中检索**（同一套 `/` `+` 语法）
- 上一处 / 下一处按 **terms** 匹配，避免 OR 查询时按钮失效

### 语料治理（活跃库硬规则）

| 规则 | 活跃库 `data/<时期>/` | 隔离区 `data/_raw_no_punct/` |
|------|----------------------|-----------------------------|
| 文字 | **简体**（`opencc t2s`） | 可保留繁体/原料 |
| 标点 | **有标点**（密度门控） | 无标点本可暂存 |
| 命名 | 文件名仅 **`书名.txt`** | 备份/原料任意 |
| 展示 | 标签 **`书名·[时代]·作者`** | 不进 `/api/periods` 检索 |
| 排除 | 目录名以 `_` 开头的路径不参与搜索 | 含 `_cache`、`_raw_no_punct` |

另：编年/会编类支持**异体字映射**（含扩展区、日文新字体如 増→增）与**语义分段**（保护 `【…】` 注释；按年号、四季、干支、引书句式断段）。

- [x] `ThreadingHTTPServer` 多线程，避免单线程卡住显示「网络错误」
- [x] `GET /api/health` · `GET /api/periods` · `GET /api/search` · `GET /api/text`
- [x] 查询解析：`parse_query` + `line_matches`（组间 AND / 组内 OR）
- [x] 搜索响应：`total`（真实总数）、`returned`、`offset`、`limit`、`has_more`、`terms`、`keywords`
- [x] 移动端：粘性顶栏、折叠「筛选范围」、触控目标、搜索框 16px
- [x] 双模式：全文 | 书名
- [x] 书组折叠 + 全局收起/展开
- [x] 分页「加载更多」（默认每页 1000，上限 5000）

### 语料与入库

- [x] 固定时期目录 + 空时期仍列出
- [x] 用户上传优先：Feishu/本地 txt/epub 清洗入库（编码探测含 GB18030）
- [x] 开放源批量：维基文库 MediaWiki API、殆知阁（daizhigev20）、Project Gutenberg（部分短书）
- [x] 标点密度审计；模型自动标点队列（`scripts/auto_punctuate.py`，CPU）
- [x] 全库/单书 `opencc t2s`；异体映射与会编/要录重分段
- [x] 辽宋金夏批次：宋史/辽史/金史、长编及拾补、会编、要录、契丹国志等（以 `data/辽宋金夏/` 为准）
- [x] 先秦等子部/经部补缺脚本（`fetch_missing_classics.py` 等）

### 运维注意（踩过的坑）

- [x] **禁止**用截断后的 `len(page)` 当 `total`（否则南宋大书占满 200 条后，金史等后排书「搜不到」）

## 源与工具

### 文本来源（开放或用户提供）

| 来源 | 用途 |
|------|------|
| **用户自备** | 购买/整理的 epub、txt（如长编、会编、要录、战国史料编年辑证等）— 优先覆盖开放抓取 |
| **中文维基文库** | 多卷标点本，MediaWiki API 分页拉取 |
| **殆知阁 daizhigev20**（GitHub） | 大覆盖 UTF-8 史部/子部等，常无标点 → 进隔离区或自动标点 |
| **Project Gutenberg** | 少数短篇公版 |
| **Wikisource 四库系** | 常无现代标点，需甄别后隔离或标点 |

> 仓库内文本为研究自用整理快照；若你 fork 公开，请自行核对版权与许可。

### 运行与处理工具

| 工具 | 用途 |
|------|------|
| **Python 3** | `http.server.ThreadingHTTPServer` 后端；入库/清洗脚本 |
| **浏览器静态页** | `static/index.html`（无构建、无前端框架） |

## 快速开始

```bash
cd /path/to/text-search
python3 -u server.py 8899
# 浏览器打开 http://127.0.0.1:8899
# 部署后改前端务必硬刷新：Ctrl+F5 / Cmd+Shift+R
```

健康检查与检索示例：

```bash
curl -s http://127.0.0.1:8899/api/health
curl -s --get \
  --data-urlencode 'q=兀术/乌珠/宗弼' \
  --data-urlencode 'periods=辽宋金夏' \
  --data-urlencode 'limit=1000' \
  http://127.0.0.1:8899/api/search
```

### API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 存活 |
| GET | `/api/periods` | 时期 + 书目 + 标签元数据 |
| GET | `/api/search?q=&mode=fulltext\|title&periods=&books=&context=&limit=&offset=` | 检索 |
| GET | `/api/text?period=&file=` | 全书行数组（阅读器） |

查询语法（正文与书内检索相同）：

- `无恤/毋恤` — 任一词  
- `无恤+伯鲁` — 同一行同时包含  
- `兀术/乌珠/宗弼` — 多名异写

```
├── logs/                  # 本地日志（gitignore）
├── README.md
└── .gitignore
```

## 数据约定

1. **文件名** = 纯书名 + `.txt`（不要把 `·[时代]·作者` 写进文件名）。  
2. **元数据** 写在 `books_meta.json`，与入库同一批改。  
3. **时期目录** 跟目录芯片（用户可要求史记放「先秦」而标签仍是西汉）。  
4. **同一书名只保留一条活跃路径**。  
5. 大体积原料与自动标点中间态放 `_raw_no_punct/`，本仓库 `.gitignore` 已排除。
